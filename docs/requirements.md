# Prefix Guard 要件仕様書

## 概要
Prefix Guardは、git commitメッセージにプレフィックスを付けることをサポートするCLIツールです。inkを使用して、美しい対話型インターフェースを提供します。

## 開発フェーズ

### Phase 1: コア機能 - プレフィックス管理（優先度: 高）
#### 1. プレフィックス定義
- プレフィックスの形式サポート
  - デフォルト形式
    - `[fix]` 形式
    - `fix:` 形式
  - カスタム形式の設定
    - プレフィックスの前後に付ける文字の設定
    - プレフィックスとタイトルの間の区切り文字の設定
- プレフィックスの説明文管理
  - 各プレフィックスの説明文設定
  - 説明文の表示機能

#### 2. プレフィックス設定管理
- 設定ファイル形式
  - JSONC（JSON with Comments）形式
  - コメントによる設定の説明
  - スキーマ定義による設定の検証
- プロジェクト固有の設定
  - `.prefix-guard.jsonc` での設定
  - デフォルトプレフィックス設定
  - プロジェクト固有の形式設定
- グローバル設定
  - ユーザーホームディレクトリでの設定
  - デフォルト設定の上書き
  - グローバルな形式設定

#### 3. 対話型プレフィックス選択
- プレフィックス一覧表示
  - 説明文付き表示
  - カテゴリ別表示
- プレフィックス検索
  - インクリメンタルサーチ
  - 説明文での検索
- プレフィックス選択
  - キーボード操作
  - マウス操作（オプション）

### Phase 2: コミット操作サポート（優先度: 中）
#### 1. コミットメッセージ作成
- プレフィックス選択後のタイトル入力
- 本文入力（オプション）
- コミットメッセージのプレビュー

#### 2. ステージングエリア管理
- 変更ファイルの一覧表示
- ステージング状態の表示
- コミット対象の選択

### Phase 3: git add -p サポート（優先度: 低）
#### 1. 変更の対話型ステージング
- 変更のプレビュー表示
- 変更の分割（split）サポート
- 変更の編集（edit）サポート

## 技術要件

### 1. 開発環境
- Node.js 16以上
- TypeScript
- ink (React for CLIs)
- git

### 2. パフォーマンス要件
- コマンド実行時の応答時間: 1秒以内
- メモリ使用量: 100MB以下

### 3. 互換性
- macOS, Linux, Windows での動作保証
- git 2.0以上との互換性
